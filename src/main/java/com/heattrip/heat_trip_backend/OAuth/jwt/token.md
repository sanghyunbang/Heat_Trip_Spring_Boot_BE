### 헷갈리는 토큰 : Oauth access 토큰과 JWT

| 종류                         | 발급 주체                      | 목적                            | 우리가 만드나?    |
| -------------------------- | -------------------------- | ----------------------------- | ----------- |
| 1️⃣ **OAuth Access Token** | Google, Kakao 등 **소셜 제공자** | 소셜 사용자 정보를 얻기 위해 사용           | ❌ 구글이 발급    |
| 2️⃣ **JWT (우리 서비스용)**      | **우리 서버(Spring Boot)**     | Flutter 앱에서 우리 API 호출할 때 인증 용 | ✅ 우리가 직접 생성 |


### 전체 흐름 개괄

1) 사용자가 Flutter 앱에서 소셜 로그인 (구글 예시)
2) 구글이 발급한 access_token을 Spring Security가 받아서 그 토큰으로 사용자 정보(json) 가져옴 (OAuth2User)
3) 사용자 정보를 기반으로 내부 자체 JWT를 발급

**내부 토큰(JWT) 생성 이유**
| 이유        | 설명                                                             |
| --------- | -------------------------------------------------------------- |
| 🔐 독립성    | 구글/카카오가 발급한 access\_token은 **우리 서버가 이해하기 어려움**                 |
| 💼 서비스 통합 | 우리 사용자 시스템(DB)과 매핑해야 함                                         |
| 🔐 보안 통제  | JWT 안에 role, 유효시간 등 **우리만의 정책**을 담을 수 있음                       |
| 📱 프론트 연동 | Flutter 앱에서 우리 API 호출 시 `Authorization: Bearer JWT` 형식으로 사용 가능 |
